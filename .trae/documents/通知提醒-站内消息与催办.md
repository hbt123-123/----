# 通知提醒 - 站内消息与催办（阶段 6）

## Context

部长驾驶舱（阶段 5）已上线，预警区已能列出逾期/临期/待审核任务，但**预警信息无法触达当事人**：干事不知道自己被催办，副部长不知道有待审核任务压在自己项目上。本期补齐「通知提醒」能力，形成「预警 → 触达 → 处理」闭环。

三件事：
1. **站内消息**：notifications.json 持久化，按 user_id 隔离
2. **逾期/临期自动提醒**：独立 CLI（`python cli.py remind`），由 systemd timer / cron 调用，零常驻（不在 Flask 进程内开线程）
3. **驾驶舱批量催办按钮**：部长/副部长一键对某类预警批量发送手动催办

约束：无新依赖（不用 APScheduler/SSE）、复用 `dashboard._classify_task`、原子写、24h/1h 双窗口去重。

### 2.2 `backend/cli.py`（新建，独立 CLI 入口）

**不用 Flask CLI**：项目用 `python app.py` 直接启动，未配 FLASK_APP；`scan_and_remind` 是纯函数，不需要 app context。Blueprint 实例化不需要 app，导入安全。

```bash
python cli.py remind                       # 扫描全部三类
python cli.py remind --type overdue        # 仅逾期
python cli.py remind --dry-run             # 只统计不写入
```

### 2.3 `backend/routes/notifications.py`（新建蓝图）

`bp = Blueprint('notifications', __name__, url_prefix='/api/notifications')`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/` | login_required | 当前用户消息，`?unread=1&limit=N&offset=M`，created_at desc |
| GET | `/unread-count` | login_required | 轻量端点，60s 轮询用，`{count: N}` |
| POST | `/<id>/read` | login_required + 归属校验 | 标记已读 |
| POST | `/read-all` | login_required | 全部已读 |
| DELETE | `/<id>` | login_required + 归属校验 | 删除单条 |
| POST | `/remind` | role_required('部长','副部长') | 批量催办，1h 窗口去重 |

```json
{ "category": "overdue|due_soon|pending_review",
  "project_id": "可选",
  "task_ids": ["可选子集"] }
```
返回：`{sent, deduped, skipped}`

副部长按 `load_project` 归属过滤可见项目；不可见项目传 project_id → 403。

## 三、提醒文案模板

### auto（from_kind='auto'，from_user='system'）

- overdue: `任务逾期提醒` / `「{proj}/{stage}/{team}/{material}」已逾期(截止 {due}),请尽快提交。`
- due_soon: `任务临期提醒` / `「{proj}/{stage}/{team}/{material}」将于 {due} 到期,请及时提交。`
- pending_review: `待审核任务提醒` / `「{proj}/{stage}/{team}/{material}」已提交待审核,请尽快处理。`

### manual（from_kind='manual'，from_user=催办人 user_id）

- overdue: `[催办] 任务逾期` / `{催办人姓名} 催办你:「{proj}/{stage}/{team}/{material}」已逾期(截止 {due}),请尽快提交。`
- due_soon: `[催办] 任务临期` / `{催办人姓名} 催办你:「...」将于 {due} 到期,请及时提交。`
- pending_review: `[催办] 待审核任务` / `{催办人姓名} 提醒你:「...」已提交待审核,请尽快处理。`

link 统一 `/projects/{project_id}`。

## 四、去重策略

| 触发方 | 窗口 | 说明 |
|--------|------|------|
| auto (cli.py) | 24h | systemd timer 每 2h 跑一次 → 同任务每天最多 1 条自动提醒 |
| manual (POST /remind) | 1h | 部长 1h 内不能对同一任务重复催办；1h 后可再催 |

manual 窗口查到的记录可能是 auto 30 分钟前刚发的 → manual 跳过（避免重复打扰）；1h 后 manual 可发，auto 下次跑时 24h 内已有 manual 记录 → auto 跳过。两者协同。

---

## 五、边界情况

| 场景 | 处理 |
|------|------|
| `assignee_id == ''`（overdue/due_soon） | 跳过，计入 `skipped_no_assignee` |
| `meta.owner_ids == []`（pending_review） | 跳过该任务，计入 `skipped_no_owner`（但 admin 仍会收到，因 _resolve_recipients 含所有部长） |
| assignee 指向已删用户 | `find_user` 返回 None → 跳过，计入 `skipped_user_missing` |
| `meta.status != '进行中'` | 整个项目跳过，计入 `skipped_project_closed` |
| admin（不在 owner_ids） | pending_review 显式纳入目标（admin 可审全部） |
| due_date 为空/解析失败 | `_classify_task` 返回 (False, False) → 自然不发 |
| 任务 status='已通过' | 不触发任何提醒 |
| 副部长传不可见 project_id | load_project 返回 403 |

### 6.6 `frontend/src/views/Dashboard.vue`（修改）

三个预警卡片 `.section-title` 各加「全部催办」按钮（逾期红/临期黄/待审核蓝，plain 风格），点击调 `remindTasks({category})`，loading 状态 + ElMessage 反馈 `已发送 N 条,跳过 M 条`。

```js
import * as notificationsApi from '@/api/notifications'
const reminding = ref({ overdue: false, due_soon: false, pending_review: false })
async function remind(category) {
  reminding.value[category] = true
  try {
    const res = await notificationsApi.remindTasks({ category })
    ElMessage.success(`已发送 ${res.sent} 条提醒${res.deduped ? `,跳过 ${res.deduped} 条(最近已发)` : ''}`)
  } finally { reminding.value[category] = false }
}
```

干事视图无此按钮（仅部长/副部长可见，`v-if="!isStaff"`）。

## 九、端到端验证

### 后端 CLI

1. 启动后端，admin 登录建项目，生成任务指派给 `u_001`，手改 `tasks.json` 中该任务 `due_date` 为过去时间
2. `python cli.py remind --dry-run` → 输出 `{sent:1, deduped:0}` 但不写文件
3. `python cli.py remind` → 写入 `notifications.json` 一条 overdue 通知
4. 立即再跑 `python cli.py remind` → `{sent:0, deduped:1}`（24h 去重）

### 待审核提醒

1. 干事上传任务文件 → status='已提交'
2. `python cli.py remind --type pending_review` → 向 `owner_ids ∪ {admin}` 各发一条

### 前端铃铛与轮询

1. 干事登录前端 → 铃铛显示未读数
2. 点铃铛 → popover 显示最近 8 条
3. 点通知项 → 跳 `/projects/:id` + badge 减 1
4. 「全部已读」→ badge 清零

### 通知中心页

1. popover 点「查看全部」→ `/notifications`
2. 切「未读」tab → 只显示未读
3. 删除按钮 → 列表项消失

### 驾驶舱批量催办

1. admin 工作台「逾期任务」卡点「全部催办」→ ElMessage `已发送 N 条`
2. 切干事账号 → 铃铛新增 `[催办] 任务逾期` 通知
3. 1h 内再点 → `已发送 0 条,跳过 N 条`

### 副部长权限边界

1. 副部长只看到自己的 owner 项目，催办只对可见项目生效
2. curl 直接 POST 不可见 project_id → 403

### 边界

- 无 assignee 任务 → `skipped_no_assignee`
- 已删用户被指派 → `skipped_user_missing`
- `meta.status != '进行中'` → 整项目跳过
