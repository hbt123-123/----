# 部长驾驶舱 — 工作台重构

## Context（背景）

项目当前已完成阶段 0-4（认证 / 成员 / 模板 / 项目 / 团队 / 任务 / 汇总），核心材料管理闭环已跑通。根据 `DEVELOPMENT.md` 待开发优先级，第一项是「**部长驾驶舱**」。

当前 [Dashboard.vue](file:///e:/project/资环创实/frontend/src/views/Dashboard.vue) 仅是静态欢迎页（你好 + 提示文案），未利用已落地的项目/任务数据。本次开发把工作台重构为**角色分层驾驶舱**：

- 部长/副部长一眼掌握：进行中项目数、待审核/逾期/临期任务数、各项目阶段进度、人员负荷、预警清单
- 干事看到：个人任务统计 + 待办列表（按截止时间排序，逾期高亮）

目标是把已有数据转化为管理价值，替代当前静态欢迎页。**不引入新依赖**（无图表库），用 `el-progress` + 纯 CSS 横条完成可视化。

## 方案

### 1. 后端：新增 dashboard 蓝图

**新建** [backend/routes/dashboard.py](file:///e:/project/资环创实/backend/routes/dashboard.py)，蓝图前缀 `/api/dashboard`。

**端点** `GET /api/dashboard`：一次性返回驾驶舱全部数据，按角色裁剪（一次请求拿全，避免前端多次拉取）。

**复用现有工具**（不重写）：
- [storage.read_json](file:///e:/project/资环创实/backend/storage.py) 读 JSON
- [config.PROJECTS_DIR / PROJECTS_INDEX / USERS_FILE](file:///e:/project/资环创实/backend/config.py) 路径常量
- [auth.load_users / to_public](file:///e:/project/资环创实/backend/auth.py) 用户脱敏
- [guards.role_required](file:///e:/project/资环创实/backend/guards.py) 权限装饰器（部长/副部长/干事均可访问，按角色裁剪返回）

**数据聚合逻辑**：
1. 读 `projects.json` 索引 → 遍历每个项目目录读 `meta.json` / `stages.json` / `tasks.json`
2. 副部长：跳过 `owner_ids` 不含自己的项目；干事：仅收集 `assignee_id == 自己` 的任务
3. 按 stage_id 分组统计任务状态；按 assignee_id 分组统计人员负荷
4. 逾期/临期判定（后端 `datetime.now()`，避免前端时区问题）：
   - **逾期**：`due_date` 非空且 `due_date < now` 且 `status != '已通过'`
   - **临期**：`now <= due_date <= now + 3 天` 且 `status not in ('已通过','已打回')`
   - `due_date` 格式 `YYYY-MM-DD HH:mm`，用 `datetime.strptime(s, '%Y-%m-%d %H:%M')` 解析；解析失败则跳过该任务的逾期判定

### 4. 前端：重构 Dashboard.vue

**重写** [frontend/src/views/Dashboard.vue](file:///e:/project/资环创实/frontend/src/views/Dashboard.vue)。

**部长/副部长视图**（按角色 `auth.role` 判断）：

1. **顶部统计卡片区**（4 张 el-card 横排，v-motion 进场动画）
   - 进行中项目（primary 蓝）/ 待审核（warning 黄）/ 逾期（danger 红）/ 临期（warning 黄）
   - 大数字 + 图标 + 标签，点击「待审核」「逾期」可滚动到对应预警区

2. **项目进度卡片网格**（el-row + el-col，每行 2 张卡）
   - 每张卡：项目名 + 状态 tag + 级别 tag + 整体进度条（`el-progress :percentage="passed/total*100"`）
   - 卡内展开各阶段：阶段名 + 截止时间 + 阶段小进度条 + 逾期红点标记
   - 点击卡片 `router.push('/projects/' + id)`

3. **人员负荷视图**（el-card 内 el-table 或纯 CSS 横条）
   - 每行：姓名 + 角色 tag + 总任务数 + 待审核数 + 逾期数（红色）
   - 用纯 CSS 横条（width 按任务数占比）可视化负荷

4. **预警区**（3 个 el-card 纵向堆叠或 tab 切换）
   - 逾期任务表（行红色背景 `--el-color-danger-light-9`）
   - 临期任务表（行黄色背景 `--el-color-warning-light-9`）
   - 待审核任务表（显示 submitted_at）
   - 每行点击跳转项目详情

**干事视图**：
- 4 张个人统计卡（我的任务 / 已提交 / 已通过 / 逾期）
- 我的待办任务列表（按 due_date 升序，逾期行红底高亮，状态用 el-tag 着色：未交 info / 已提交 warning / 已通过 success / 已打回 danger）

**样式约定**（复用现有 [main.css](file:///e:/project/资环创实/frontend/src/styles/main.css) 色彩变量）：
- 主色 `--el-color-primary` #1664ff / success #1ebf6f / warning #bd7e00 / danger #f65159
- 文本 `--el-text-color-primary` #0c0d0e / secondary #737a87
- 卡片标题 15px / 600，与 Projects.vue 的 `.title` 一致
- 统计卡大数字 28px / 600

**复用**：
- `el-card` `el-progress` `el-tag` `el-table` `el-empty` `el-row` `el-col`
- `@element-plus/icons-vue` 图标（DataAnalysis / Warning / Clock / Document 等）
- `@vueuse/motion` v-motion 进场动画（与现有 Dashboard 一致）
- `useAuthStore` 角色判断

   - 项目进度卡片（每项目含阶段进度条）

   - 建项目（选大创模板）→ 导入团队 → 选阶段生成任务 → 部分上传 → 部分审核通过

6. 干事登录 → 仅看到个人统计卡 + 我的待办列表

   - 无项目：统计卡显示 0，项目区显示 `el-empty`「暂无项目」
